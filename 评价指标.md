### 目标函数（Objective Function）

| **序号**     | **指标名称**                                                 | **描述**                                                     |
| ------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **1**        | **长期平均系统成本 (Overall Long-Term Average Cost)**        | $$\bar{C} = \lim_{T\to\infty}\frac{1}{T}\sum_{t=1}^{T}\Big(E(t)+\lambda_{\text{cloud}}U_{\text{cloud}}(t)+\lambda_{\text{sw}}S(t)\Big)$$ |
| **选择原因** | 这是李雅普诺夫优化框架 (OLMA) 的**核心优化目标** ，旨在在保证网络稳定性的前提下，最小化总能量消耗、云使用成本和切换成本的加权和 2。**最小化此指标**直接体现了算法的长期经济效益和能效。 |                                                              |

### 评价指标 (Key Performance Indicators, KPIs)

| **序号**     | **指标名称**                                                 | **逻辑层次**        | **描述**                                                     |
| ------------ | ------------------------------------------------------------ | ------------------- | ------------------------------------------------------------ |
| **3**        | **平均端到端时延 (Average End-to-End Delay)**                | **服务质量**        | 统计任务的排队、传输、计算及回传时延的总和 。是衡量 VEC 服务质量 (QoS) 的**最关键指标**，特别是对于安全关键应用 。 |
| **选择原因** | 李雅普诺夫方法通过控制队列积压 ($\bar{Q}$) ，**隐式地控制了时延** 。同时，原文实验结果直接强调了该指标的降低。 |                     |                                                              |
| **5**        | **队列稳定性 (Queue Stability)**                             | **稳定性/鲁棒性**   | $$\bar{Q} = \frac{1}{T}\sum_{t}\sum_{i}Q_i(t)$$ 即平均队列长度。 |
| **选择原因** | 它是李雅普诺夫优化**机制成立的基石** 。只有 $\bar{Q} < \infty$ (即队列稳定)，才能证明算法在随机、动态环境下的**长期可靠性**和**时延受控性** 。 |                     |                                                              |
| **2**        | **成本–时延权衡曲线 ([O(1/V), O(V)] Trade-off)**             | **理论验证**        | 通过改变控制参数 $V$ 来观察 $\bar{C}$ 与 $\bar{D}$ 之间的系统性权衡关系。 |
| **选择原因** | 这是**李雅普诺夫在线控制框架的标志性理论成果** 。绘制此曲线是**验证算法理论最优性**（即 $C \le C^{opt} + B/V$ ）和**控制参数有效性**的必要手段。 |                     |                                                              |
| **18**       | **单时隙决策时延 (Decision Time per Slot)**                  | **复杂度/可落地性** | 测量算法在每个时间槽内计算出联合决策所需的时间。             |
| **选择原因** | VEC 是一个**高动态、实时**系统 。算法需要**低复杂度** 才能在单个时隙的严格时间约束内得出决策，实现真正的**实时操作** 。该指标体现了 OLMA 算法 (结合了 MDO 和 SCRA) 在实际部署中的可行性。 |                     |                                                              |

**总结：**

- **目标函数：** 长期平均系统成本
- **3 个主要 KPIs：** 平均端到端时延、队列稳定性、成本–时延权衡曲线
- **备选/补充 KPI：** 单时隙决策时延
